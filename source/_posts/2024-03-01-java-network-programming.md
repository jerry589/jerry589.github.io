# Java网络编程基础知识详解

本文整理了Java网络编程的基础概念和核心知识点，包括Web基础概念、I/O流处理、多线程编程等内容。

## Web基础概念

### URI、URL和URN

**URI (Uniform Resource Identifier)** - 统一资源**标识**符，是一种用来**标识**某一网络上**资源**的抽象标识。

包括以下两种形式：

**URL (Uniform Resource Locator)** - 统一资源**定位**符，是URI的一种。提供资源在网络中的具体**位置**

**URN (Uniform Resource Name)** - 统一资源**名称**，同样是URI的一种。为资源提供一个持久的、易于理解的名称

**URL格式：**
```
[协议]://[user:password@]domain[:port]/path[?query][#fragment]
```

**URN格式：**
```
urn:[命名空间]:[标识符]
```

**例子：** `urn:isbn:0-486-27557-4`

**片段（Fragment）**：在URI、URL的上下文中，是指一个用于标识网页内部特定部分的标识符。片段的格式非常简单，它以井号（`#`）开始，后面紧跟着一个或多个字符。例如：`#section2`

### HTML、SGML、XML、MIME

- **SGML**：一种功能非常强大且复杂的标记语言，它被设计为一种元语言，可以用来创建其他标记语言，如HTML和XML
- **HTML**：用于创建网页和网页应用的标准标记语言，主要目的是**定义网页的结构**和呈现方式
- **XML**：可扩展标记语言，用于**描述数据结构和内容**的标记语言，可以自定义
- **MIME**：多用途邮件扩展，用来说明文件类型的标准，由类型（type）和子类型（subtype）组成，例如"text/html"

### CGI和Applet

**CGI（Common Gateway Interface）**：通用网关接口，是一种标准，用于定义Web服务器和外部程序之间的接口，实现动态页面，可以用任意编程语言来写。

**Applet**：在Java平台上运行的小型程序，通常嵌入在HTML页面中，并通过Web浏览器在用户的计算机上执行。可以提供交互式的客户端应用程序功能，如游戏、动画、交互式表单等。

## 输入/输出（I/O）

网络程序的很大一部分工作就是简单的输入输出，也就是从一个系统到另一个系统移动字节。Java把所有的输入/输出以流的形式进行处理。

### 流的概念

**流**是指**连续的、单向**的数据传输的一种抽象，即由源到目的地通信路径传输的一串字节。
- **发送数据流**的过程称为**写**
- **接受数据流**的过程称为**读**
- 读取流返回-1的时候表示已经读取完数据了

### 字节流（Byte Streams）

字节流主要用于处理二进制数据，它们**以字节为单位**进行操作。

- `InputStream`：用于读取字节数据的抽象基类，使用read方法
- `OutputStream`：用于写入字节数据的抽象基类，使用write方法
- 常用类：`FileInputStream`和`FileOutputStream`，用于文件的字节读写操作

### 字符流（Character Streams）

字符流主要用于处理文本数据，它们**以字符为单位**进行操作。

- `Reader`：用于读取字符数据的抽象基类
- `Writer`：用于写入字符数据的抽象基类
- 常用类：`FileReader`和`FileWriter`，用于文件的字符读写操作

### 缓冲流和刷新流

**缓冲流**：一个个字节写入的效率太低，因此要设置缓冲区，等累计到了一定字节再转发到目的地。
- `BufferedOutputStream`
- `BufferWriter`

**刷新流**：keep-alive开启的时候会希望及时得到响应，如果缓冲区的太少发不出去，将会产生对方收不到数据的情况，所以在关闭所有流之前，应当立即刷新它们。

**注意**：最后应该使用`close()`方法关闭流，它将会释放该流的资源。

### I/O操作注意事项

1. **数据类型转换**：尽管read()方法读取的是字节，但是由于Java没有无符号字节的数据类型，所以数据以整型返回。需要进行数据类型转换，int范围是-128-127，需要加256变成0-255的无符号字节。

2. **多字节读取**：读取的时候要考虑多字节读取，单字节太慢了，使用重写的read方法将读取的字节先存到一个数组里`read(byte[] input)`。

## 多线程编程

### 基本概念

1. **进程（Process）**：CPU从硬盘中读取一段程序到内存中，该执行程序的实例就叫做进程。进程是执行程序的一次执行过程，是动态的概念。

2. **线程**：进程中的实际运作的单位，是进程的一条流水线，是程序的实际执行者，是最小的执行单位。

3. **何时需要多线程**：
   - 程序需要同时执行两个或多个任务
   - 程序需要实现一些需要等待的任务时，如用户输入、文件读写操作、网络操作、搜索等
   - 需要一些后台运行的程序时

### 线程的使用

- `run()`方法是线程的执行体，即线程要**执行的任务**
- 调用`start()`方法启动线程，启动的时候会自动调用一次`run()`
- `t.run()`不会启动线程，只是普通的调用方法
- `t.start()`方法会启动一个分支线程，在JVM中开辟一个新的栈空间

### 线程同步与锁

当使用多个线程来访问同一个数据时，这个数据在被一个线程访问完成前不允许被其他线程访问，这就叫**同步**。

Java语言提供`synchronized`关键字来实现同步：

**同步代码块：**
```java
synchronized(someobject){
    // 省略代码
}
```

**同步方法：**
```java
public synchronized void method() {
    // 方法体
}
```

### 线程相关异常

`InterruptedException`是Java中的一个异常，用于指示线程在执行过程中被中断。当线程正在执行诸如`sleep`、`wait`、`join`等操作时，如果另一个线程通过调用该线程的`interrupt`方法强制中断它，那么正在执行的操作会抛出`InterruptedException`。

## 其他重要概念

### Sun包

一般说来，"sun"之类的包并不包含在Java平台的标准中，它与操作系统相关，在不同的操作系统（如Solaris，Windows，Linux，Mac等等）中的实现也各不相同。

### 多态

多态指的是同一个行为具有多个不同表现形式或形态的能力：

- **方法重载**：多个名称相同但参数列表不同
- **方法覆盖**：在类的继承关系中，子类可以覆盖父类中的方法，提供自己的实现

## 总结

Java网络编程涉及多个重要概念，从基础的Web概念到I/O流处理，再到多线程编程，每个部分都是构建网络应用程序的重要基础。理解这些概念对于开发高效、稳定的网络应用程序至关重要。

在实际开发中，需要特别注意：
- 正确处理I/O流的开启和关闭
- 合理使用多线程避免资源竞争
- 理解不同流类型的适用场景
- 掌握线程同步机制防止数据不一致

这些知识点将为后续深入学习Java网络编程打下坚实的基础。
