# JAVA网络编程

### web概念


<!-- more -->
**URI (Uniform Resource Identifier)** - 统一资源**标识**符，是一种用来**标识**某一网络上**资源**的抽象标识。

包括以下两种形式

**URL (Uniform Resource Locator)** - 统一资源**定位**符，是URI的一种。提供资源在网络中的具体**位置**

**URN (Uniform Resource Name)** - 统一资源**名称**，同样是URI的一种。为资源提供一个持久的、易于理解的名称

URL：\[协议]://\[user:password@]domain\[:port]/path\[?query]\[#fragment]

URN:`urn:[命名空间]:[标识符]`

例子：`urn:isbn:0-486-27557-4`\
片段（Fragment），在URI、URL的上下文中，是指一个用于标识网页内部特定部分的标识符

片段的格式非常简单，它以井号（`#`）开始，后面紧跟着一个或多个字符，这些字符用来表示文档内部的一个特定部分。例如：

\#section2

### CGI,applet

CGI（Common Gateway Interface，通用网关接口）是一种标准，用于定义Web服务器和外部程序之间的接口,实现动态页面，可以用任意编程语言来写，就比如PHP里面用来连接数据库的那种就叫CGI

Applet是一种在Java平台上运行的小型程序，它通常嵌入在HTML页面中，并通过Web浏览器在用户的计算机上执行。Applets可以提供交互式的客户端应用程序功能，如游戏、动画、交互式表单等。它们在90年代末和21世纪初非常流行，但随着Web技术的发展，尤其是HTML5和JavaScript能力的增强，applets的使用已经大幅减少。

### **输入/输出（I/O）**

网络程序的很大一部分工作就是简单的输入输出，也就是从一个系统到另一个系统移动字节，很大程度上读服务器发送的数据与读取文件没有什么不同

Java把所有的输入/输出以流的形式进行处理，这里的流是指**连续的、单向**的数据传输的一种抽象，即由源到目的地通信路径传输的一串字节\*\*。发送数据流**的过程称为**写\*\*，**接受数据流**的过程称为**读**。

读取流返回-1的时候表示已经读取完数据了

##### 字节流（Byte Streams）

字节流主要用于处理二进制数据，它们**以字节为单位**进行操作。在Java中，最基本的字节流类是`InputStream`和`OutputStream`，所有的字节流都是这两个抽象基类的子类。

*   `InputStream`：用于读取字节数据的抽象基类。（我吸收输入）使用read方法写入read(int a)or (byte\[] data) 写入的是一个无符号字节

*   `OutputStream`：用于写入字节数据的抽象基类。（我输出）使用write输出&#x20;

一些常用的字节流类包括`FileInputStream`和`FileOutputStream`，分别用于从**文件读取数据和向文件写入数据。**

##### 字符流（Character Streams）

字符流主要用于处理文本数据，它们**以字符为单位**进行操作。字符流会考虑字符编码，因此可以自动处理字符到字节的转换。基本的字符流类是`Reader`和`Writer`。

*   `Reader`：用于读取字符数据的抽象基类。

*   `Writer`：用于写入字符数据的抽象基类。

常用的字符流类有`FileReader`和`FileWriter`，用于文件的字符读写操作。

##### 缓冲流

一个个字节写入的效率太低了，因此要设置缓冲区，等累计到了一定字节再转发到目的地

##### 刷新流

keep-alive开启的时候**会希望及时得到响应**，如果缓冲区的太少发不出去，将会产生对方收不到数据的情况，所以在关闭所有流之前，应当立即刷新它们

注意：最后应该使用close()方法关闭流，它将会释放该流的资源

###### 了解

开启是由于多次反复开启会浪费资源，而一次TCP连接就可以传送多次数据，所以会希望及时得到响应好进行下一块环节

HTTP守护进程是指在客户端没有再发送请求一段时间以后自动关闭该连接，算作连接结束

如果动态服务请求多就不建议keep-alive

假如这个时候有100个用户访问服务器，且每个用户的MaxKeepAliveRequests\=5(也就是浏览器的请求并发数是5)，，此时需要的http守护进程数就是100\**5\=500个，一个http守护进程消耗是5MB内存的话，就是500*5\=2500MB\=2.5G。这内存的消耗也太大了。

### 线程

1.  进程（Process） CPU从硬盘中读取一段程序到内存中，该执行程序的实例就叫做进程 一个程序如果被CPU多次被读取到内存中，变成多个独立的进程。将程序运行起来，我们称之为进程。进程是执行程序的一次执行过程，他是动态的概念。进程存在生命周期，也就是说程序随着程序的终止而销毁。进程之间是通过TCP/IP端口实现交互的。

2.  线程是进程中的实际运作的单位，是进程的一条流水线，是程序的实际执行者，是最小的执行单位。通常在一个进程中可以包含若干个线程。线程是CPU调度和执行的最小单位。

3.  最起码，现在的java程序中至少有两个线程并发，一个是 垃圾回收线程，一个是 执行main方法的主线程。

4.  何时需要多线程

    1.  程序需要同时执行两个或多个任务

    2.  程序需要实现一些需要等待的任务时，如用户输入、文件读写操作、网络操作、搜索等。

    3.  需要一些后台运行的程序时。

5.  很多线程并发都是模拟出来的，真正的多核是指多个服务器，模拟出来的多线程是计算机串行多个进程，交叉运行，只不过因为速度快感觉不出来

6.  `run()`方法是线程的执行体，即线程要**执行的任务**。在`run()`方法中编写的代码将定义线程的行为。

7.  调用start()方法启动线程,启动的时候会自动调用一次run()

注意 ：

1.t.run()不会启动线程，只是普通的调用方法而已。不会分配新的分支栈。（这种方法就是单栈程） t.start()方法的作用是 ：**启动一个分支线程**，在JVM中开辟一个新的栈空间，这段代码任务完成之后，瞬间就结束了。 这段代码的任务只是为了开启一个新的栈空间，只要新的栈空间开出来，start()方法就结束了。线程就启动成功了。**启动成功的线程会自动调用run方法**，并且run方法在分支栈的栈底部（压栈）。run方法在分支栈的栈底部，main方法在主栈的栈底部。run和main是平级的。&#x20;

\`\`\`2.InterruptedException`是 Java 中的一个异常，属于`java.lang`包。它用于指示线程在执行过程中被中断。当线程正在执行诸如`sleep`、`wait`、`join`等操作时，如果另一个线程通过调用该线程的`interrupt`方法强制中断它，那么正在执行的操作会抛出`InterruptedException\`。

#### 线程同步与锁

在卖票程序中可能碰到一种意外，同一张票号被打印两次或多次，也可能打印票号为0或是负数（多窗口卖同一张票）

当使用多个线程来访问同一个数据时，这个数据在被一个线程访问完成前不允许被其他线程访问。这就叫**同步**。

为了**避免**多线程共享资源发生冲突的情况的发生，只要在线程使用资源时给该资源**上一把锁**就可以了

Java语言提供synchronized(同步的)关键字

两种同步形式，即同步方法和同步代码块。

语法格式为：

synchronized(someobject){

     ……//省略代码

}
