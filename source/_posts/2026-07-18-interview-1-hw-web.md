# 渗透测试面试系列一：HW护网、安全设备与前期准备

<!-- more -->

## HW护网

### 护网担任过什么角色？

监控、研判、处置、应急、溯源、带队。

### 护网前期准备工作有哪些？

**资产梳理**：

- 甲方提供资产范围，做资产收集梳理，检查是否存在未备案的资产
- 梳理出核心资产、边界资产
- 外网暴露资产：IP、域名、端口、运行服务、中间件、口令强度、源码泄露
- 远程接入点：VPN、RDP、VNC、远程终端管理系统
- 内网资产：IP、端口、开放服务、系统版本、口令强度、域控安全
- 重点关注资产：域控、邮件、OA、堡垒机、运维人员机器、自动化运维

**人员安排**：

- 值守人员安排：监控 → 研判 → 处置 → 应急 → 溯源

**安全加固**：

- 基线检查、漏洞补丁、升级版本、增强口令、邮箱拦截、代码审计、白名单访问
- 边界资产能否下线处置，提出合理化建议
- 外网高危端口采用白名单访问
- 网络区域划分（DMZ区、办公终端、核心业务、上下游公司、供应链接入区）

**安全设备**：

- 设备类型：蜜罐、WAF、网络层、终端层、日志汇总态势
- 安全覆盖范围：主域/子域/子公司

**其他**：

- 由于子公司没有安全服务团队，对资产进行渗透测试
- 使用安全厂商设备安全扫描
- 上报漏洞，提供修复方案/临时修复方案，跟踪修复进度情况（指定修复时间）
- 加强人员安全意识：防钓鱼、防弱口令
- 寻找互联网暴露面切入点

## 安全设备

### 常见WAF产品

腾讯WAF、长亭WAF、天融信WAF。

### IPS入侵防御系统

IPS一般也是在网络层旁路，可以理解为具备阻断能力的IDS，是IDS的升级版（也有IDS检测到攻击通知阻断设备执行阻断动作的设备联动模式），可以覆盖网络层和应用层。

代表产品：NIPS腾讯天幕。

### IDS入侵检测系统

IDS工作在网络层，旁路部署，通过抓取和分析网络流量来发现攻击。

代表产品：

- 奇安信天眼
- 奇安信NGSOC
- 腾讯御界NTA

### HIDS主机入侵检测系统

主机安全层面，部署在服务器上的入侵检测系统。

代表产品：

- 青藤云设备
- 奇安信椒图
- 腾讯云镜

### 蜜罐

蜜罐就是利用 JSONP 跨域，类似 CSRF 漏洞，如果攻击者在未开启无痕模式的状态下访问网站，且常用网站的 Cookie、身份验证未失效的情况下，蜜罐网站代码中使用 JSONP 跨域访问某些 API，获取个人信息、ID、手机号等。

> 只要攻击者使用浏览器访问了蜜罐，并且浏览器缓存中有蜜罐配置的利用漏洞获取用户信息的网站 Cookie，那么蜜罐即可利用 JSONP 或者 XSS 漏洞来获取攻击者的个人信息，有了姓名、ID、手机号等信息后，溯源红队也变得更加容易。

## 工具基础

### SQLMap 扫 HTTPS 会出现什么问题？

使用 SQLMap 注入 HTTPS 的网站时，如果没有进行任何处理则会报错，连接不了目标。原因是 SSL 证书的原因，SQLMap 不能自动伪造 SSL 证书。

解决方法：

1. 添加 `--force-ssl` 参数
2. 走本地代理端口访问

### 指纹识别工具的原理是什么？

通过匹配响应中的特定字符串、HTTP头、Cookie、页面结构、文件MD5、特殊路径等特征，与已知的指纹库进行比对，从而识别目标的CMS、框架、中间件、服务及版本信息。

## 蓝队反制手段

1. **蜜罐**：利用 JSONP/XSS 获取攻击者身份信息
2. **对攻击目标进行反渗透**：IP定位、IP端口扫描、Web站点渗透
3. **应用漏洞挖掘&利用**：菜刀、Goby、Xray、蚁剑
4. **ID → 社交特征关联**：百度/Google搜索 ID 在各大平台的痕迹
5. **钓鱼网站 → 后台扫描、XSS盲打**
6. **木马文件 → 同源样本关联 → 敏感字符串特征检测**
7. **APP蜜罐**

针对蜜罐的反制：HFish 蜜罐曾存在未授权读取信息漏洞及存储 XSS 漏洞，默安科技幻阵蜜罐也曾存在未授权访问漏洞，利用蜜罐自身漏洞来反制蜜罐。
