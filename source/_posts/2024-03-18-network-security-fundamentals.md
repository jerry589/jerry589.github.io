# 网络安全基础知识学习笔记

本文整理了网络安全领域的基础概念和核心技术，涵盖了安全术语、Web架构、攻击技术、防护措施等重要内容，为网络安全学习和实践提供全面的参考。

## 基础安全术语

### 漏洞分类

**0day零日漏洞**：还没有补丁的安全漏洞，也称为零时差漏洞。

**1day漏洞**：漏洞被公开披露后，相关软件或系统的开发者在第一天内发布修复程序的漏洞。

### 攻击链条

**POC（Proof of Concept）**：概念验证，常指一段漏洞证明的代码。

**EXP（Exploit）**：利用，指利用系统漏洞进行攻击的动作。

**Payload**：有效载荷，指成功exploit之后，真正在目标系统执行的代码或指令。

**Shellcode**：shell代码，是Payload的一种，由于其建立正向/反向shell而得名。

**攻击流程**：发现漏洞 → 攻击 → 攻击后的措施（POC → EXP → Payload/Shellcode）

### 安全工具和概念

**蜜罐**：吸引攻击者的诱饵，主要作用包括：
- 欺骗攻击者，吸引攻击者进行攻击
- 捕获和分析攻击行为
- 了解攻击工具与方法

**后门**：不会被发现的恶意传输通道或会话

**WebShell**：专门针对网站的后门

**木马vs病毒**：
- 木马：主要针对系统，控制电脑，窃取信息
- 病毒：本质是破坏性的，目的是搞破坏

### 测试方法

**黑盒测试**：只给目标，什么都没有，自己寻找渗透方式

**白盒测试**：能找到相关源码，通过源码分析达到渗透目的

**社会工程学**：根据人的行为习惯猜测密码，通过工具或脚本解析密码

**撞库**：查询网上泄露的数据库寻找目标信息

## Web基础架构

### 服务器架构

**LAMP架构**：Linux + Apache + MySQL + PHP

**LNMP架构**：Linux + Nginx + MySQL + PHP

### Web服务器

**Apache**：
- 开源Web服务器软件
- 主要用于处理HTTP请求
- 配置文件：`httpd.conf`
- 默认网页目录：`htdocs`
- 只能解析HTML内容，需要PHP解析引擎处理PHP文件

**Nginx**：
- 高性能HTTP和反向代理服务器
- 擅长处理高并发请求
- 资源消耗较低

**IIS**：
- 微软的Web服务器
- 与Apache都是广泛使用的Web服务器软件

### 应用服务器

**Tomcat**：
- 开源Java Servlet容器
- 用于运行Java Web应用程序
- 处理Java语言编写的Servlet和JSP页面

### 容器化技术

**容器**：轻量级虚拟化技术，与宿主机共享内核

**Docker**：开源应用容器引擎，用于开发、交付和运行应用程序

**沙盒**：安全机制，限制程序权限和访问资源的能力

## 网络通信和代理

### Shell连接

**正向Shell**：控制端主动发起连接去连接被控制端

**反向Shell**：被控制端主动连接控制端
- 实战中多采用反向shell
- 被控制端主动向外发送的数据包通常不会被拦截
- 避免防火墙、入侵防御系统等阻拦

### 代理技术

**反向代理**：
- 在服务器自身不知道的情况下进行代理
- 多用于负载超量的前提下
- 实现负载均衡
- 例：Nginx作为前置反向代理监听80端口，Tomcat改为监听8081端口

**跳板**：将请求发送到服务器或节点，以跳板身份帮助搜索并返回结果

## 身份认证和会话管理

### HTTP无状态问题解决方案

**Cookie维持**：
- 服务端验证成功后设置cookie给客户端
- 每个HTTP请求都自动带数据给服务端
- 风险：用户可以F12查看cookie，容量限制

**Session维持**：
- 服务端建立session会话，返回sessionid给客户端
- 实际账号密码在服务端，安全性更高
- 可以存储对象
- 缺点：集群服务器情况下sessionid信息不好扩展

**Token维持**：
- 服务端生成令牌返回给客户端
- 客户端带上Token进行请求
- 无需再次带上用户名和密码

### JWT（JSON Web Token）

**组成部分**：
1. **Header（头部）**：
```json
{
  "alg":"HS256",
  "typ":"JWT"
}
```

2. **Claims（声明）**：包含用户信息和权限

3. **Signature（签名）**：使用密钥对头部和声明进行加密

**优势**：
- 将用户状态分散到客户端
- 减轻服务端内存压力
- 状态存储在客户端而非服务器端

## 安全攻击技术

### RCE（远程命令执行）

**定义**：让攻击者直接向后台服务器远程注入操作系统命令或代码，控制后台系统

**常见场景**：
- 路由器、防火墙、入侵检测设备的web管理界面
- 文件上传下载功能
- 系统命令执行接口

### XSS（跨站脚本攻击）

**反射型XSS**：不持久，诱导别人点击链接

**存储型XSS**：持久链接，一直在留言板等地方

**DOM型XSS**：全部由前端JavaScript代码生成

### 注入攻击

**SQL注入**：通过在输入中插入恶意SQL代码来攻击数据库

**XPath注入**：类似SQL注入，操纵XPath查询来访问不应该访问的数据

### 劫持攻击

**点击劫持**：
- 使用透明iframe覆盖目标网页
- 视觉欺骗用户点击隐藏的恶意内容
- 通过视觉伪装吸引用户点击

**DNS劫持**：
- 篡改域名解析结果
- 使域名指向错误的IP地址
- 劫持用户访问到假冒网址

## 加密算法

### 对称加密

**DES**：
- 64位明文加密成64位密文
- 密钥长度56位
- 支持多种模式（ECB、CBC等）

**AES**：
- Advanced Encryption Standard
- DES的进阶版本
- 更高的安全性

### 非对称加密

**RSA**：
- 基于大质数相乘容易，质因数分解困难的数学原理
- 公钥加密，私钥解密
- 用于密钥交换和数字签名

## 系统安全

### Windows安全

**NTFS权限**：
- NT File System Permissions
- 对文件和文件夹的访问权限控制
- 权限级别：system > administrator > users

**注册表**：
- 存储操作系统和程序的配置信息
- 主要分支：HKEY_CLASSES_ROOT、HKEY_CURRENT_USER等

### 进程监控

**恶意进程判断**：
```cmd
netstat -abnov
-a 显示监听端口以及连接
-b 显示链接的文件名
-n 数字形式显示地址和端口号
-o 显示PID进程标识符
```

### PowerShell安全

**执行策略**：
- Restricted：脚本不能运行（默认）
- RemoteSigned：本地脚本可运行，网络下载的需要签名
- AllSigned：仅受信任发布者签名的脚本可运行
- Unrestricted：允许所有脚本运行

## 开发框架和工具

### MVC架构

**Model（模型）**：数据结构和业务逻辑

**View（视图）**：用户界面展示

**Controller（控制器）**：连接Model和View的中间层

### 模板引擎

**作用**：
- 分离逻辑和展示
- 动态内容生成
- 提高代码可重用性

**常见模板引擎**：Jinja2、Mustache、Thymeleaf等

### 数据库技术

**Redis**：
- 基于键值对的NoSQL数据库
- 高性能（微秒级读写）
- 高并发支持
- 常用作缓存层

**结构化vs非结构化**：
- 结构化：如MySQL，每条记录字段相同
- 非结构化：NoSQL，记录字段不固定

## 版本控制

### Git基础

**分布式vs集中式**：
- 集中式：SVN、CVS，需要服务器支持
- 分布式：Git，每个电脑都是服务器

**Git组成框架**：
1. **Workspace（工作区）**：存放项目文件
2. **Index/Stage（暂存区）**：准备下次提交的改动
3. **Repository（本地仓库）**：所有提交的历史版本
4. **Remote（远程仓库）**：其他电脑上的仓库

## 安全防护

### WAF（Web应用防火墙）

**作用**：保护Web应用的特殊防火墙

### 安全域划分

**原则**：
- 技术层面隔离（防火墙、隔离网络）
- 业务流程分析
- 减少安全威胁传播

### 漏洞管理

**CVE**：Common Vulnerabilities & Exposures，公共漏洞和暴露

**CWE**：软件缺陷的类型分类

## 总结

网络安全是一个综合性的领域，涉及多个层面：

1. **基础知识**：掌握安全术语、攻击链条、测试方法
2. **技术架构**：理解Web架构、服务器配置、容器化技术
3. **攻击技术**：了解各种攻击手段和原理
4. **防护措施**：掌握安全防护技术和最佳实践
5. **工具使用**：熟练使用各种安全工具和框架
6. **持续学习**：跟上安全威胁的发展趋势

网络安全需要理论与实践相结合，在学习基础知识的同时，要通过实际操作加深理解，建立完整的安全防护体系。
