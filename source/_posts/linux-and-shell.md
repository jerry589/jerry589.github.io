# Linux 和 Shell 学习笔记

## 操作系统是什么


<!-- more -->
操作系统是计算机的一个重要组成部分，它是计算机上的一个系统软件，负责管理和控制计算机的硬件和软件资源

| **操作系统类型** | **描述** | **常见版本/产品** |
| :----------: | :--------------------------------: | :---------------------------------------------: |
| Windows | 由微软公司开发的个人电脑操作系统，提供直观的图形用户界面 | Windows 10, Windows 8.1, Windows 7 |
| macOS | 由苹果公司开发的Mac电脑专用操作系统，设计优雅，用户体验良好 | macOS Catalina, macOS Mojave, macOS High Sierra |
| Linux | 自由和开放源码的类Unix操作系统，稳定性和安全性高 | Ubuntu, CentOS, Debian, Fedora |
| UNIX | 功能强大、稳定可靠的操作系统，常用于服务器和高性能计算 | AIX, Solaris, FreeBSD, OpenBSD |
| Android（移动端） | 由谷歌公司开发的开放源代码移动操作系统，广泛应用于智能手机和平板电脑 | Android 11, Android 10, Android 9 Pie |
| iOS（移动端） | 由苹果公司开发的专有移动操作系统，只能在苹果公司的设备上运行 | iOS 14, iOS 13, iOS 12 |

## linux系统目录结构

![](<linux and https://gitee.com/jerry798/xcximg/raw/master/img/c8986410-ec16-11ee-933d-9d65ad8ed90a.jpeg?v=1\&type=image>)

| **目录** | **描述** |
| :-------------: | :---------------------------------------------------------------------: |
| **/boot** | 存放**启动Linux时**使用的内核文件、连接文件及镜像文件。 |
| **/etc(翻译：等等)** | 存放系统所需的**配置文件**和子目录列表。更改这里的文件可能会导致系统无法启动。 |
| **/lib** | 存放基本代码库，类似于Windows中的DLL文件。当编译程序时，如果程序需要使用某个库中的函数或类，编译器会查找这个库文件，比如C++的函数 |
| **/sys** | 集成proc、devfs和devpts文件系统的信息，反映内核设备树。 |
| **/bin** | 存放最常用的程序和指令，ls等等。 |
| **/sbin** | 存放只有系统管理员能使用的程序和指令。 |
| **/dev** | 存放Linux的外部设备。 |
| **/media** | 存放识别后的其他设备，如U盘、光驱等。 |
| **/mnt** | 临时挂载其他文件系统的目录。 |
| **/run** | 临时文件系统，存储系统启动以来的信息。 |
| **/lost+found** | 系统非法关机后存放一些文件的目录。 |
| **/tmp** | 存放临时文件的目录。 |
| **/root** | 系统管理员的用户主目录。 |
| **/home  \~** | 用户的主目录，**以用户的账号命名**。 |
| **/usr** | 存放用户的应用程序和文件，类似于Windows的**Program Files**目录。 |
| **/usr/bin** | 存放系统用户使用的应用程序与指令。 |
| **/usr/sbin** | 存放超级用户使用的高级管理程序和系统守护程序。 |
| **/usr/src** | 内核源代码的默认放置目录。 |
| **/var** | 存放经常修改的数据，如程序运行的**日志文件**。 |
| **/proc** | 虚拟目录，管理内存空间，是系统内存的映射。 |
| **/opt** | 存放额外安装的软件。 |
| **/srv** | 存放服务启动后需要提取的数据（不用服务器则为空） |

当计算机打开电源后，首先是**BIOS开机自检**，按照BIOS中设置的启动设备（通常是硬盘）来启动。

操作系统接管硬件以后，首先读入 **/boot 目录下的内核文件**。

运行init 进程（系统所有进程的起点）

**sync**将内存中的数据与磁盘中的数据同步，确保数据的一致性和持久性

## Linux 文件基本属性(用户，组，权限)

![](<linux and https://gitee.com/jerry798/xcximg/raw/master/img/9b3a9c10-ec37-11ee-933d-9d65ad8ed90a.jpeg?v=1\&type=image>)

rwx分别代表读、写、执行，-为没有权限

通常三个为一组，第一个是单个为一组的文件类型，有多种，d表示目录

chown (change owner) ： 修改**所属用户与组**。

```bash
chown [-R] 所有者:属组名 文件名          
[root@www ~]# chown bin install.log
[root@www ~]# ls -l
-rw-r--r--  1 bin  users 68495 Jun 25 08:53 install.log
现在这个文件就是bin用户的了
```

chmod (change mode) ： 修改**用户的权限**。

我们可以使用数字来代表各个权限，各权限的分数对照表如下：

r:4

w:2

x:1

每种身份(owner/group/others)各自的三个权限(r/w/x)分数是需要累加的，例如当权限为： **-rwxrwx---** 分数则是：

owner = rwx = 4+2+1 = 7

group = rwx = 4+2+1 = 7

others= --- = 0+0+0 = 0

```bash
chmod [-R] xyz 文件或目录
```

## Linux 磁盘管理

Linux 磁盘管理好坏直接关系到整个系统的性能问题。

Linux 磁盘管理常用三个命令为 **df**、**du** 和 **fdisk**。

**df**（英文全称：disk free）：列出文件系统的整体磁盘使用量

**du**（英文全称：disk used）：检查磁盘空间使用量

**fdisk**：用于磁盘分区

**磁盘格式化**

磁盘分割完毕后自然就是要进行文件系统的格式化，格式化的命令非常的简单，使用 `mkfs`（make filesystem） 命令。

语法：

```bash
mkfs [-t 文件系统格式] 装置文件名
```

**磁盘检验**

fsck（file system check）用来检查和维护不一致的文件系统。

若系统掉电或磁盘发生问题，可利用fsck命令对文件系统进行检查。

语法：

```bash
fsck [-t 文件系统] [-ACay] 装置名称
```
