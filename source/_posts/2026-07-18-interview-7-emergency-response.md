# 渗透测试面试系列七：应急响应、溯源反制、日志分析与主机排查

<!-- more -->

## 应急处置

### 应急处置标准流程

1. **收集信息**：收集告警信息、客户反馈信息、设备主机信息等
2. **判断类型**：安全事件类型判断（钓鱼邮件、WebShell、爆破、中毒等）
3. **控制范围**：隔离失陷设备
4. **分析研判**：根据收集回来的信息进行分析
5. **处置**：根据事件类型进行处置（进程、文件、邮件、启动项、注册表排查等）
6. **输出报告**

### 攻防演练/重保过程中发生真实攻击的应急流程

1. **攻击确认**：研判分析告警内容，判断是否为真实攻击/误报。真实攻击封禁 IP，判断是否利用成功
2. **事件类型确定**：根据告警确定事件类型——勒索/挖矿/蠕虫/钓鱼/文件上传/WebShell流量/反弹Shell/外联/敏感操作
3. **范围确定**：根据安全设备确定受影响资产/主机设备范围
4. **抑制扩散/横向**：在甲方允许且对业务不造成重大影响的情况下，对已沦陷/受影响设备进行断网阻断或不出网的防火墙策略，或 Web 业务下线
5. **主机排查**：根据事件类型上机排查，清除 WebShell、结束目标/外联/恶意进程。再对目标可能存在的权限维持手段进行排查清除
6. **内网影响范围**：基于 HIDS 主机防御系统，查看失陷主机有没有进行内网横向渗透阶段，关注同网段主机设备是否存在被攻击/失陷可能
7. **事件分析**：基于流量态势感知 IDS/主机防御系统 HIDS（或Web日志、主机日志），找出红方攻击时间段、攻击 IP、代理 IP、影响范围、攻击流程、漏洞利用点、漏洞分析/钓鱼邮件分析/恶意程序，攻击点复现
8. **修复加固**：出具修复/加固方案
9. **输出报告**：红方攻击流程/利用阶段，蓝方应急响应/阻断情况/漏洞分析/修复加固/业务上线

## 主机排查

### Linux 入侵排查流程

1. 检查账号安全
2. 检查账号中的危险信息
3. 检查用户执行过的命令（history）
4. 检查异常端口（netstat）
5. 检查进程的状况（ps）
6. 检查启动脚本
7. 检查计划任务（crontab）
8. 检查开机自启服务
9. 查找可疑文件
10. 查看系统日志

### Windows 入侵排查流程

1. 账号安全
2. 启动项 / 策略组 / 计划 / 服务
3. 端口
4. 进程
5. 查看事件日志

### Linux 系统日志

日志默认存放位置：`/var/log/`

| 日志文件 | 说明 |
|---------|------|
| `/var/log/cron` | 记录了系统定时任务相关的日志 |
| `/var/log/cups` | 记录打印信息的日志 |
| `/var/log/dmesg` | 记录了系统在开机时内核自检的信息，也可以使用 `dmesg` 命令直接查看 |
| `/var/log/maillog` | 记录邮件信息 |
| `/var/log/message` | 记录系统重要信息的日志，系统出现问题时首先要检查的日志 |
| `/var/log/btmp` | 记录错误登录日志，二进制文件，使用 `lastb` 命令查看 |
| `/var/log/lastlog` | 记录系统中所有用户最后一次登录时间，使用 `lastlog` 命令查看 |
| `/var/log/wtmp` | 永久记录所有用户的登录、注销信息，同时记录系统启动、重启、关机事件，使用 `last` 命令查看 |
| `/var/log/utmp` | 记录当前已经登录的用户信息，使用 `w`、`who`、`users` 等命令查询 |
| `/var/log/secure` | 记录验证和授权方面的信息，SSH 登录、su 切换用户、sudo 授权等 |

## 溯源反制

### 针对攻击 IP 的溯源思路

1. 确定 IP 的归属：VPS？国内云服务器？运营商？企业 IP？
2. 钓鱼邮件分析
3. 对 IP 反渗透

### 溯源攻击画像——补全攻击者信息

- IP
- 邮箱
- 微信
- QQ
- 身份证号
- 学校
- ID
- 公司

### 通过 ID 有哪些溯源思路

- 百度 → ID 在哪发过文章、发表过评论 → 个人博客
- Google → ID 在哪发过文章、发表过评论 → 个人博客
- CSDN
- GitHub
- 各大安全社区
- 微信
- QQ

### 钓鱼邮件如何分析/处置

1. 沙箱/虚拟机/恶意行为分析/查找外网 IP
2. 查找钓鱼历史/微步查询/丢给二线处理
3. 丢虚拟机分析
4. 丢沙箱分析
5. 丢微步查询
6. 钓鱼样本分析，哈希值对比，找出目标团队

### 钓鱼邮件防沙箱手段

攻击者会在邮件或文档中做反沙箱检测：延时执行、环境检测、用户交互检测。

### 如何通过邮箱知道对方的 IP

分析邮件头的 `Received:` 字段，逐条追溯发送路径中的 IP 地址。

### 蓝队常用反制手段

1. **蜜罐**：JSONP/XSS 获取攻击者身份信息
2. **对攻击目标进行反渗透**：IP 定位、IP 端口扫描、Web 站点渗透
3. **应用漏洞挖掘&利用**：菜刀、Goby、Xray、蚁剑
4. **ID → 社交特征关联**
5. **钓鱼网站 → 后台扫描、XSS 盲打**
6. **木马文件 → 同源样本关联 → 敏感字符串特征检测**
7. **APP 蜜罐**

### 针对蜜罐的反制手段

HFish 蜜罐曾存在未授权读取信息漏洞及存储 XSS 漏洞，默安科技幻阵蜜罐也曾存在未授权访问漏洞。利用蜜罐自身漏洞来反制蜜罐。

### 拿到肉鸡的权限，从哪些方面寻找攻击者痕迹？

- 命令历史（history/bash_history）
- SSH 登录日志
- 上传/下载文件记录
- 定时任务
- 后门程序中的 C2 地址
- 进程列表中的异常连接

## 内存马

### 原理

内存马是无文件 WebShell。在 Web 组件或者应用程序中注册一层访问路由，访问者通过这层路由来执行控制器中的代码。就像 MVC 架构，即通过路由访问控制器。

服务器上不会存在需要连接的 WebShell 脚本文件。

### 内存马类型

- Java：Filter 型、Servlet 型、Listener 型、Valve 型、Spring Controller/Interceptor 型、Agent 型
- PHP：OPcache 马
- .NET：IIS Module 马

### 内存马如何查杀

1. 重启/关机 → 内存马就掉了
2. 任务管理器查看内存消耗
3. Java：使用 Arthas 排查 Filter/Listener/Servlet 注册情况、JMX MBean 检测
4. 内存马查杀工具

### 如何判断有没有进行横向

1. 内网主机安全设备查看同网段主机是否被爆破、扫描、漏洞利用等
2. 判断横向（没有隧道/代理工具时，对内网扫描，排查存在隐患的主机）

### 如何证明恶意的宏被启用和点击了？

检查注册表：

```
HKEY_LOCAL_MACHINE\USERDAT\Software\Microsoft\Office\<VERS>\<PROGRAM>\Security\Trusted Documents\TrustRecords
```

只寻找最后四个字节是 `FF FF FF 7F` 的值，这些文件已经启用了宏程序。

## 常用命令速查

### Linux 命令

| 命令 | 作用 |
|------|------|
| `netstat` | 查看端口 |
| `ps` | 查看进程列表 |
| `grep` | 文件中搜索字符串 |
| `history` | 历史操作记录 |
| `ls -al` | 查看隐藏文件+权限 |
| `last` | 查看最近登录信息 |
| `crontab -l` | 检查定时任务 |
| `uname -a` | 查看版本当前操作系统内核信息（详细） |
| `cat /proc/version` | 查看版本当前操作系统内核信息 |

### Linux 管道符

| 符号 | 说明 |
|------|------|
| `;` | 无视前面出错，前面语句执行完再执行后面 |
| `\|` | 无视前面出错，显示后面语句执行结果 |
| `\|\|` | 如果前面出错，则执行后面语句 |
| `&` | 无视前面出错，前面执行完再执行后面 |
| `&&` | 如果前面出错，则不执行后面语句 |

### Windows 命令

| 命令 | 作用 |
|------|------|
| `tasklist` | 显示当前运行的进程列表 |
| `taskkill /F /PID 6400` | 根据 PID 结束进程 |
| `taskkill /IM javaw` | 根据进程名结束进程 |
| `wmic` | 查看指定进程信息 |
| `quser` | 查看用户是否在线 |
| `query user` | 当前登录账号 |
| `net user` | 查看当前所有用户 |
| `net user cc2 cc2 /add` | 添加用户 |
| `net user cc2$ cc2 /add` | 添加隐藏用户 |
| `net localgroup administrators cc2 /add` | 添加用户至管理员组 |
| `reg query HKLM\SAM\SAM\Domains\Account\Users\Names` | 查看隐藏用户 |
| `net localgroup` | 显示当前所有组 |
| `net localgroup Administrators` | 查看管理员组成员 |

### Windows 管道符

| 符号 | 说明 |
|------|------|
| `&` | 直接运行，不管成功/失败 |
| `&&` | 前面运行成功才运行第二个 |
| `\|` | 前面运行成功才运行第二个，显示后面的结果 |
| `\|\|` | 运行不成功才运行第二个 |
