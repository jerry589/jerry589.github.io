## 免杀

### Go 语言 Shellcode 免杀原理

1. Go 编译出的二进制文件体积较大，天然不太容易被特征码定位
2. 使用 garble 混淆编译（`garble -literals -tiny build`）
3. Shellcode 使用 XOR/AES 加密存储，运行时解密
4. 延迟 syscall 实现，不过 Go 的 syscall 封装层

### Windows Defender 防御机制原理与绕过

**Defender 防御机制**：

- 特征码扫描
- 行为监控（AMSI）
- 云查杀（MAPS）
- 实时保护

**绕过思路**：

1. AMSI 绕过（patch `AmsiScanBuffer`）
2. 代码混淆/加密
3. 进程注入
4. 白加黑利用

### Shellcode 免杀思路

1. 加密：XOR、AES、RC4 加密 Shellcode，运行时解密
2. 混淆：拆分、编码、变形 Shellcode
3. 加载方式：进程注入、回调执行、APC 注入
4. 绕过行为检测：Syscall 直通、API Unhooking

### 提取内存 Hash 被查杀如何绕过？

1. 使用 procdump 导出 lsass.dmp，离线用 mimikatz 读
2. 修改 mimikatz 源码重新编译
3. 使用不落地的方式（反射加载）

### PowerShell 免杀怎么制作？

1. 编码绕过（Base64 编码、自定义 XOR 编码）
2. 拆解执行（payload 分片拼接）
3. AMSI 绕过
4. 反射加载 .NET Assembly
5. 不落地执行（`IEX (New-Object Net.WebClient).DownloadString`）

### 如何把 Shellcode 嵌入到正常 EXE 中？

1. **资源段注入**：将 Shellcode 写入 PE 文件资源段，修改入口点
2. **进程镂空（Process Hollowing）**：启动正常程序并挂起，镂空其代码后注入 Shellcode
3. **DLL 劫持**：将 Shellcode 放在被劫持的 DLL 中
4. **节区扩展**：在 PE 文件中新增节区存放 Shellcode

## 钓鱼

### 邮件钓鱼的准备工作有哪些？

1. 确定邮件钓鱼的形式：链接 or 文件
2. 收集目标相关邮箱
3. 编写钓鱼邮件文案（结合节日/热点事件更有效）
4. 匿名邮箱
5. 木马免杀测试、钓鱼站点搭建

### 如何伪造钓鱼邮箱？会面临什么问题？

- 伪造显示名称（From: CEO Name \<spoofed@gmail.com\>）
- 使用相似域名
- SPF/DKIM/DMARC 检测问题
- 邮件网关拦截

### 钓鱼邮件防沙箱手段

攻击者会在恶意文档/链接中做反沙箱检测——延时执行、环境检测、用户交互检测等，确保只在真实用户环境中触发恶意行为。

## 红队

### 红队高频打点漏洞

1. Apache Shiro 相关漏洞
2. Fastjson 漏洞
3. Log4j
4. 上传漏洞
5. 边界网络设备资产 + 弱口令
6. Spring、蓝凌 OA、0day、常见 CMS、弱口令

### 绕过 CDN 获取真实 IP

1. 多地/全球 Ping
2. DNS 解析记录
3. 域名历史解析记录
4. 网络测绘引擎 Fofa/Shodan
5. 子域 C 段
6. ICO 图标 MD5 值（Shodan 搜索 `http.favicon.hash:`）
7. PHP 探针
8. 邮箱邮件服务器的真实 IP

### 403 访问/目录突破

1. 修改 Host
2. 扩展名绕过
3. 覆盖请求 URL
4. Referer 标头
5. FUZZ 下级目录

### 站库分离怎么拿 WebShell

1. 如果上传点在前端 Web 服务器 → 写 WebShell 到 Web 目录
2. 如果数据库服务器可直接连接 → 通过 SQL 写文件（INTO OUTFILE）
3. 利用 SSRF 打数据库

### 如何判断蜜罐

1. 蜜罐插件检测
2. 百度 API、常见 API 检测
3. 检查系统运行时间、登录记录、用户文件数量
4. 尝试出站连接，看是否受限

### 绕过实战 WAF

**SQL 注入 Bypass**：

1. FUZZ 确认拦截了什么函数
2. 空白字符/内联注释绕过
3. 填充垃圾数据（WAF 存在长度限制）
4. 分块传输（`Transfer-Encoding: chunked`）
5. 参数污染（HPP）

**文件上传 Bypass**：参考面试系列三。

**XSS Bypass**：大小写、编码、更换标签/事件。

### 隐藏 CS 流量

1. **Malleable C2 Profile** 定制（伪装正常 Web 流量）
2. **域前置**（Domain Fronting）
3. **CDN 中转**
4. **DNS/ICMP 隧道**
5. **JA3 指纹对抗**

### CS 上线时转到会话显示是 SSH

这可能是因为 Beacon 的通信协议或管道配置问题，检查 CS 的 Listener 配置和 Payload 类型。

### SQLMap 工作原理

SQLMap 先在目标系统上写入一个拥有上传功能的木马，并对这个木马的内容进行 16 进制编码。这个木马名是由 `tmp` 加上五个随机字符组成的。然后再通过这个文件上传一个 PHP 木马文件，这个木马文件可以用来执行命令，并且可以将结果返回出来，在退出后会自动删除 Shell。

## 代码审计

### 审计流程

1. 环境搭建
2. 先进行黑盒再审计：查看每个功能点存在的转义拦截，查看用户可控的入口点，危险函数

### 审计案例

- **PHP 框架 CMS**：尝试后台文件上传 Getshell，弱口令进入后台，尝试所有上传/更新功能，图片裁剪功能
- **黑白盒/刷卡系统**：Log 泄露/用户名/密码，后台管理界面

### 审计一个 CMS 的上传功能时，你会关注哪些函数？

PHP 中：`move_uploaded_file`、`copy`、`rename`、`file_put_contents`、`fwrite`，以及后缀名校验逻辑（白名单/黑名单）、MIME 校验、文件头校验、文件名处理。

## 高级话题

### 红蓝带队经验

- 国家 HW / 高级蓝队：协调、任务调度、工作总结
- HW 前期准备有哪些：制度应急响应方案、自身掌握技能、了解甲方网络安全结构（网络架构、基线检查、排查资产、排除低级错误/自检、模拟红队攻击、生产/办公划分）
- 应对 0day：为安全产品编写正则规则

### 讲一下你的渗透测试流程

根据个人渗透经验判断，**重点在于信息收集**：

1. 信息收集（子域名、端口、指纹、敏感文件、社工信息）
2. 漏洞发现（自动化扫描 + 手动分析）
3. 漏洞利用（Web漏洞→Getshell→提权）
4. 内网渗透（信息收集→横向移动→域控）
5. 权限维持
6. 清理痕迹
7. 输出报告

### 小程序的渗透和普通渗透的差异

渗透过程不变，依旧是抓包修改参数渗透。不同点是小程序会将包下载到本地，可以使用逆向还原工具反编译。

### 你通过 Java 站入侵了 Windows，你什么权限？

取决于 Java 中间件以什么用户身份运行。通常 Tomcat 等中间件以 SYSTEM 或普通管理员权限运行。如果是 SYSTEM 权限则直接是最高权限；如果是普通管理员还需提权。

### 渗透过程优先关注的漏洞

1. 优先看弱口令/未授权访问（最低成本最高收益）
2. 其次是 Nday RCE（Struts2/Shiro/Fastjson/Log4j）
3. 文件上传
4. SQL 注入

### 如何判断目标操作系统

1. 大小写检测：Windows 大小写不敏感，Linux 大小写敏感
2. Ping TTL 值：
   - Windows：128
   - Linux：64
   - Unix：255
   - 路由器：255
   （注：TTL 值可修改，仅供参考）

### HTTP 参数污染（HPP）

由于现行的 HTTP 标准没有提及在遇到多个输入值给相同的参数赋值时应该怎样处理，而且不同的网站后端做出的处理方式是不同的，从而造成解析错误。可用来绕过 WAF。

### Host 碰撞

带 Host 的请求直接到反代服务器的 IP，如果反代服务器上面的对应 Host 配置还在，就会把请求转发到后面即内网中的对应 Host 业务服务器上，导致网站的域名删除了 A 记录的情况下也能访问/直接访问内网业务系统。

### 手工判断目标系统

1. 大小写检测：Windows 大小写不敏感，Linux 大小写敏感
2. Ping 默认返回值判断：

| 系统 | 默认 TTL |
|------|---------|
| Windows NT/2000 | 128 |
| Windows 95/98 | 32 |
| Unix | 255 |
| 路由器 | 255 |
| Linux | 64 |
| Win7 | 64 |
