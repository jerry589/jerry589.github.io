# Java网络编程学习笔记

本文整理了Java网络编程相关的核心概念和技术要点，涵盖了Web基础概念、I/O流处理、多线程编程等重要内容。

## Web基础概念

### URI、URL和URN

**URI (Uniform Resource Identifier)** - 统一资源**标识**符，是一种用来**标识**某一网络上**资源**的抽象标识。

包括以下两种形式：

- **URL (Uniform Resource Locator)** - 统一资源**定位**符，是URI的一种。提供资源在网络中的具体**位置**
- **URN (Uniform Resource Name)** - 统一资源**名称**，同样是URI的一种。为资源提供一个持久的、易于理解的名称

**URL格式：**
```
[协议]://[user:password@]domain[:port]/path[?query][#fragment]
```

**URN格式：**
```
urn:[命名空间]:[标识符]
```

例子：`urn:isbn:0-486-27557-4`

片段（Fragment）在URI、URL的上下文中，是指一个用于标识网页内部特定部分的标识符，以井号（`#`）开始。

### 标记语言体系

- **SGML**：一种功能强大且复杂的标记语言，被设计为一种元语言，可以用来创建其他标记语言，如HTML和XML
- **HTML**：用于创建网页和网页应用的标准标记语言，主要目的是**定义网页的结构**和呈现方式
- **XML**：可扩展标记语言，用于**描述数据结构和内容**的标记语言，可以自定义
- **MIME**：多用途邮件扩展，用来说明文件类型的标准，由类型（type）和子类型（subtype）组成，例如"text/html"

### CGI和Applet

**CGI（Common Gateway Interface）**：通用网关接口，是一种标准，用于定义Web服务器和外部程序之间的接口，实现动态页面。

**Applet**：在Java平台上运行的小型程序，通常嵌入在HTML页面中，通过Web浏览器在用户计算机上执行，可提供交互式的客户端应用程序功能。

## Java I/O流处理

### 流的概念

Java把所有的输入/输出以流的形式进行处理，流是指**连续的、单向**的数据传输的一种抽象，即由源到目的地通信路径传输的一串字节。

- **写**：发送数据流的过程
- **读**：接受数据流的过程
- 读取流返回-1表示已经读取完数据

### 字节流（Byte Streams）

字节流主要用于处理二进制数据，**以字节为单位**进行操作：

- `InputStream`：用于读取字节数据的抽象基类，使用read方法
- `OutputStream`：用于写入字节数据的抽象基类，使用write方法
- 常用实现：`FileInputStream`和`FileOutputStream`

### 字符流（Character Streams）

字符流主要用于处理文本数据，**以字符为单位**进行操作，会考虑字符编码：

- `Reader`：用于读取字符数据的抽象基类
- `Writer`：用于写入字符数据的抽象基类
- 常用实现：`FileReader`和`FileWriter`

### 缓冲流和刷新

**缓冲流**：为了提高效率，设置缓冲区，累计到一定字节再转发到目的地
- `BufferedOutputStream`
- `BufferedWriter`

**刷新流**：在keep-alive开启时，需要及时得到响应，应当在关闭所有流之前立即刷新它们。最后使用`close()`方法关闭流释放资源。

### 注意事项

1. Java没有无符号字节数据类型，read()方法返回整型，需要进行数据类型转换
2. 建议使用多字节读取提高效率：`read(byte[] input)`
3. 合理使用缓冲区和及时刷新流

## 多态

多态指同一个行为具有多个不同表现形式或形态的能力：

- **方法重载**：多个名称相同但参数列表不同的方法
- **方法覆盖**：子类覆盖父类中的方法，提供自己的实现

## 多线程编程

### 基本概念

1. **进程（Process）**：CPU从硬盘读取程序到内存中的执行实例
2. **线程**：进程中的实际运作单位，是程序的实际执行者，是最小的执行单位
3. Java程序至少有两个线程：垃圾回收线程和执行main方法的主线程

### 何时需要多线程

- 程序需要同时执行两个或多个任务
- 程序需要实现需要等待的任务（用户输入、文件读写、网络操作等）
- 需要后台运行的程序

### 线程的使用

1. **run()方法**：线程的执行体，定义线程要执行的任务
2. **start()方法**：启动线程，会自动调用run()方法

**重要注意事项：**
- `t.run()`不会启动线程，只是普通方法调用
- `t.start()`会启动新的分支线程，开辟新的栈空间
- start()方法不能重复调用，否则抛出`IllegalThreadStateException`

### 线程实现方式

1. **继承Thread类**
2. **实现Runnable接口**（推荐）
3. **实现Callable接口**

推荐使用Runnable接口，因为线程池只能放入实现Runnable或Callable的线程。

### 线程同步与锁

**问题**：多线程访问共享资源时可能发生冲突（如卖票程序中同一张票被多次售出）

**解决方案**：使用synchronized关键字实现同步

**同步形式：**

1. **同步代码块**：
```java
synchronized(someobject){
    // 同步代码
}
```

2. **同步方法**：在方法声明中加入synchronized关键字

**死锁问题**：多个线程互相等待对方释放资源

**解决死锁**：通过线程通信，协调线程之间的运行

## 总结

Java网络编程涉及多个重要概念：

1. **Web基础**：理解URI/URL/URN、标记语言体系、CGI和Applet
2. **I/O流**：掌握字节流和字符流的使用，合理使用缓冲和刷新机制
3. **多线程**：理解进程和线程概念，掌握线程创建和同步机制
4. **最佳实践**：使用Runnable接口、线程池管理、避免死锁

这些知识为Java网络编程和并发编程奠定了坚实的基础。
