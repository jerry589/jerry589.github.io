### 基础名词及概念

1.蜜罐定义：吸引攻击者的诱饵

蜜罐在网络安全中的主要作用包括以下几个方面：

**欺骗攻击者**：

蜜罐通过模拟易受攻击的主机或服务，**吸引攻击者进行攻击**，从而达到欺骗的目的。这种技术本质上是对攻击方进行欺骗的技术，通过布置一些作为诱饵的主机、网络服务或者信息，诱使攻击方对它们实施攻击。

**捕获和分析攻击行为**：蜜罐能够**捕获攻击流量与样本**，发现网络威胁、提取威胁特征，通过对网络攻击流量的捕获，进而捕获到目标恶意代码，并进行进一步的**恶意代码行为分析**

**了解攻击工具与方法**：通过蜜罐技术，可以了解攻击方所使用的工具与方法，推测攻击意图和动机，这有助于防御方清晰地了解他们所面对的安全威胁，并通过技术和管理手段进行应对。

0day零日漏洞

也可以称为零时差漏洞，通常是指**还没有补丁的安全漏洞。**

1day 漏洞

当“零日攻击”被发现并公开后，没有补丁的一段时间内（通常时间会很短，一般为1天），根据习惯这个漏洞会被称为 1day 漏洞。1day 漏洞是指漏洞被公开披露后，相关软件或系统的开发者在第一天内发布修复程序的漏洞。虽然漏洞已经被公开，但软件的开发者在第一天内就发布了修补程序，以减少攻击者利用漏洞的机会。

### 基础架构

**1.PHP语言**：

PHP是一种服务器端的脚本语言，用于在服务器上预处理数据，然后发送到客户端。

PHP常用于创建动态网站，可以与数据库交互，实现内容的实时更新。

**2.网站类型**：

静态网站：由保存在服务器上的文件组成，内容不会变化。

动态网站：与数据库相连，根据用户行为动态生成内容。

**3.静态与动态页面解析**：

静态页面解析：浏览器通过URL获取IP地址，访问服务器上的Apache进程，解析URI找到网页并返回给浏器。

动态页面解析：在Apache服务器中，页面由PHP引擎处理，可能涉及数据库查询。

注：Apache只能解析html的内容，所以需要有个PHP解析引擎

**4.apache服务器配置**：

`bin`：存放二进制可执行文件的目录。

`conf`：配置文件目录。

`lib`：核心文件夹，存放库文件。

`htdocs`：Apache默认的网页文件夹。

`modules`：模块目录，可以加载到配置文件中。

5.**网络站点设置**：

修改`httpd.conf`文件中的`DocumentRoot`来设置网站的**根目录**。

使用`ServerName`和`Listen`指令来设置网站的域名和监听端口

Servername localhost:80

listen进行监听listen 80可以进行更改

**PHP与Apache集成**：

PHP作为Apache的一个模块加载，通过`LoadModule`指令实现。

使用`AddType`指令将PHP文件关联到Apache，以便Apache知道如何处理`.php`文件。

**C语言与PHP**：

在操作系统中，是指一个时间段中有几个程序都处于已启动运行到运行完毕之间， 且这几个程序都是在同一个处理机上运行，但**任一个时刻点上只有一个程序在处理机上运行。**

### 一些概念

**Apache**：Apache是一个开源的Web服务器软件，主要用于处理HTTP请求，提供Web内容。它是一个服务器软件，可以独立运行。Apache也可以被视为一种中间件，因为它提供Web服务器服务，帮助处理客户端的HTTP请求，并将请求转发给后端的应用程序。

**nginx**：nginx是一个高性能的HTTP和反向代理服务器，主要用于处理静态请求和反向代理。它特别擅长处理高并发请求，并且资源消耗较低。nginx可以与Apache等其他Web服务器一起使用，以提供更加全面的服务。

**Tomcat**：Tomcat是一个开源的Java Servlet容器，用于运行Java Web应用程序。它是Apache软件基金会的一部分。Tomcat主要用于处理Java语言编写的Servlet和JSP页面，是Java Web应用程序的运行环境。

**容器**：容器是一种轻量级的虚拟化技术，它允许开发者打包他们的应用以及依赖包到一个轻量级、可移植的容器中。容器与宿主机的操作系统共享内核，因此相比虚拟机，容器更加轻量级，启动和停止速度更快。

**容器化**：容器化是将应用程序及其依赖打包到一个容器中，以便在不同的环境中（如开发、测试、生产）可以一致地运行。容器化使得应用程序的部署和迁移更加容易和高效。

**Servlet**：Servlet是Java语言编写的小程序，用于处理客户端的请求，并生成响应。它是Java EE（现在称为Jakarta EE）的一部分，主要用于创建动态Web内容。Servlet通常在Tomcat等Java应用服务器上运行。

**沙盒**：沙盒是一种安全机制，用于限制程序的权限和访问资源的能力，以防止恶意软件对系统造成损害。沙盒机制在容器技术中广泛使用，以提高安全性。

**Docker**：Docker是一个开源的应用容器引擎，用于开发、交付和运行应用程序。它允许开发者打包他们的应用以及依赖包到一个轻量级、可移植的容器中。Docker可以管理多个容器，实现容器之间的协调和通信，这对于构建复杂的分布式系统非常有用。

**联系**：

*   **Servlet和Tomcat**：Servlet是Java EE的一部分，而Tomcat是运行Servlet的容器。

*   **容器和Docker**：Docker是一个容器化平台，用于创建和管理容器。

*   **中间件和Apache/Tomcat**：有时候一个东西可以既作为中间件，承担系统和应用软件的连接又作为容器

*   **沙盒和容器**：容器技术通常采用沙盒机制，以提高安全性。

*   中间件主要强调的是系统软件和应用软件之间的沟通和连接，容器就是给应用组件提供环境(web组件就php，asp这些)等等

### 反向代理

在**服务器自身不知道的情况下进行代理**，多用于负载超量的前提下，一个服务器的端口撑不住，又不想让用户改变端口。

让 Nginx 作为一个前置的反向代理, 监听在 80 端口上; 而第一个 tomcat 则躲到幕后, 同时它也不再监听 80 端口(需要让给 Nginx), 而改为监听一个其它没有被使用的端口, 比如 8081, 然后让 Nginx 转发请求给它处理.使得负载均衡。

### 正向shell和反向shell

\
正向shell：控制端主动发起连接去连接被控制端\
反向shell：被控制端主动连接控制端\
在实战中，大多数采用反向shell，因为正向shell有很多因素导致连接失败，\
比如说硬件设备有防火墙，入侵防御系统等，还有网站防火墙，端口占用，权限不足等场景，特别是硬件设备如果你正向连接被防火墙拦截导致打草惊蛇，后期攻击相当繁琐。\
反向shell：而被控制端主动向外发送的数据包通常都不会被拦截。

### git

集中式与分布式的区别 除了git还有svn、cvs这样的版本控制系统，它们的区别在于一个是分布式一个是集中式

集中式就是svn和csv这样的版本控制系统，分布式是git

区别在于集中式的版本控制系统每次在写代码时都需要从**服务器**中拉取一份下来，并且如果服务器丢失了，那么所有的就都丢失了，你本机客户端仅保存当前的版本信息，换句话说，集中式就是把代码放在一个服务器上集中管理，你的所有回滚等操作都需要服务器的支持。

分布式的区别在于，**每个人的电脑都是服务器**，当你从主仓库拉取一份代码下来后，你的电脑就是服务器，无需担心主仓库被删或者找不到的情况，你可以自由在本地回滚，提交，当你想把自己的代码提交到主仓库时，只需要合并推送到主仓库就可以了，同时你可以把自己的代码新建一份仓库分享给其它人。&#x20;

### Git的基本组成框架：

1.  **Workspace（工作区）**：这是你存放项目文件的地方，也就是你平常工作的地方，所有文件都放在这里。

2.  **Index / Stage（暂存区）**：在Git中，还有一个准备下一次提交的改动的暂存区，最早叫stage，现在新版本已经改成index。这个区域存放了你使用**gitadd**命令提交的文件描述信息，它位于`.git`目录下的`index`文件中。,和git stash一样

3.  **Repository（仓库区/本地仓库）**：是仓库代码，所有的提交都在这里，Git会保存好每一个历史版本，存放在仓库区。

4.  **Remote（远程仓库）**：只能是别的电脑上的仓库，即服务器仓库。你可以使用`git clone`命令将远程仓库克隆到本地，或者使用`git push`将本地的改动推送到远程仓库。

### 异步操作

异步操作是指程序中的一种执行方式，其中某些任务可以**独立于主程序的执行流程**进行。这意味着，当你启动一个异步操作时，你的程序不需要停下来等待这个操作完成，而是可以继续执行其他任务。一旦异步操作完成，它会通过某种机制（如事件、回调函数或Promises）通知主程序。

异步操作的一个典型例子是网络请求。当你的程序向服务器请求数据时，它不需要等待服务器的响应才能继续执行其他代码。一旦服务器响应，程序可以通过回调函数或其他方式来处理这个响应。

### 回调函数

回调函数是一种特殊的函数，它被设计为在其他代码执行完成后被调用。在编程中，回调函数通常用于处理异步操作完成后的结果，或者在某个事件触发时执行特定的逻辑。

回调函数可以作为参数传递给其他函数，这样当异步操作完成或者某个事件被触发时，可以执行这个预先定义好的函数。这允许程序在不阻塞主执行流程的情况下，处理异步事件。

举个例子：

```javascript
function fetchData(url, callback) {
  // 模拟一个异步网络请求
  setTimeout(() => {
    const data = "从" + url + "获取的数据";
    callback(data); // 当异步操作完成时，调用回调函数
  }, 1000);
}

function processData(data) {
  console.log(data);
}

// 使用回调函数
fetchData('https://example.com', processData);
```

在这个例子中，`fetchData`函数执行一个模拟的异步操作，并在操作完成后调用`processData`函数。`processData`函数就是回调函数，它在异步操作完成后被调用，用来处理数据。、

### MVC

（Model-View-Controller）是一种软件设计模式，用于将应用程序的业务逻辑（Model）、用户界面（View）和控制逻辑（Controller）分离开来。这种分离使得开发过程更加模块化，更易于管理和维护。以下是MVC三个主要组件的简要说明：

1.  **Model（模型）**：

    代表应用程序的数据结构和业务逻辑。

    负责数据的存储、检索和处理。

    可以包括数据库连接、数据验证和数据操作等。

2.  **View（视图）**：

    代表应用程序的用户界面。

    负责展示数据（来自Model）并以用户可交互的方式呈现。

    通常由模板引擎生成，可以包含HTML、CSS和JavaScript等。

3.  **Controller（控制器）**：

    作为中间层，连接Model和View。

    负责接收用户输入，调用Model进行处理，并将结果传递给View进行展示。

    可以处理用户请求、更新Model状态和选择适当的View进行渲染。

MVC模式的工作流程通常如下：

1.  用户通过视图界面发送请求。

2.  控制器接收请求并调用相应的模型进行处理。

3.  模型处理完毕后，将结果返回给控制器。

4.  控制器将模型返回的数据传递给视图。

5.  视图将数据渲染为用户界面。

MVC模式的优点包括：

*   **分离关注点**：将数据、业务逻辑和界面展示分离，使得各个部分可以独立开发和维护。

*   **可重用性**：模型和视图可以独立于控制器被重用。

*   **可测试性**：由于模块化，各个组件可以单独进行测试。

*   **灵活性**：视图可以不依赖于模型的变化，控制器也可以轻松地替换视图和模型。

MVC模式被广泛应用于Web应用开发中，许多流行的Web框架如Ruby on Rails、Django、Laravel、Spring MVC等都采用了MVC架构。此外，MVC也是许多桌面应用和其他类型软件的常见设计模式。

### 劫持攻击

点击劫持攻击通常涉及到以下几个关键点：

视觉欺骗，攻击者使用一个透明的 iframe 覆盖在目标网页上，由于 iframe 是透明的，用户看不到底层的实际内容，从而在执行操作时，以为是点击了顶层的内容，但实际上是点击了底层 iframe 中的内容。

视觉伪装，为了进一步迷惑用户，攻击者通常会在其创建的网页上放置一些吸引用户的元素，如游戏、视频播放器等，当用户点击这些看似无害的区域时，实际上触发的是隐藏在其下的目标网站中的敏感操作。

用户交互，当用户尝试点击覆盖层上的按钮或链接时，实际上他们点击的是下面的 iframe 中目标网站上的按钮或链接。然后会执行相应的恶意操作，如提交表单、跳转链接等，从而达到攻击者的目的。

### DNS劫持

　　DNS劫持又叫做域名劫持，指攻击者利用其他攻击手段，篡改了某个域名的解析结果，使得指向该域名的IP变成了另一个IP，导致对相应网址的访问被劫持到另一个不可达的或者假冒的网址，从而实现非法窃取用户信息或者破坏正常网络服务的目的。

　　DNS的作用是把网络地址对应到真实的计算机能够识别的网络地址，以便计算机能够进一步通信，传递网址和内容等。

### IDS入侵检测系统

IDS（intrusion detection system）入侵检测系统是一种对网络传输进行**即时监视**，在发现可疑传输时发出警报或者采取**主动反应措施**的网络安全设备。它与其他网络安全设备的不同之处便在于，IDS是一种积极主动的安全防护技术。在很多中大型企业，政府机构，都会布有IDS。我们做一个比喻——假如防火墙是一幢大厦的门锁，那么IDS就是这幢大厦里的**监视系统**。一旦小偷进入了大厦，或内部人员有越界行为，只有实时监视系统才能发现情况并发出警告。

 

专业上讲IDS就是依照一定的**安全策略**，对**网络、系统的运行状况进行监**视，尽可能发现各种攻击企图、攻击行为或者攻击结果，以保证网络系统资源的机密性、完整性和可用性。与防火墙不同的是，IDS入侵检测系统是一个**旁路监听设备**，没有也不需要跨接在任何链路上，无须网络流量流经它便可以工作。因此，对IDS的部署的唯一要求就是：IDS应当挂接在所有所关注流量都必须流经的链路上。

IDS在交换式网络中的位置一般选择为：尽可能靠近攻击源，尽可能靠近受保护资源。

##### **入侵检测系统的架构**

事件产生器：它的目的是从整个计算环境中获得事件，并向系统的其他部分提供此事件。

事件分析器：分析数据，发现危险、异常事件，通知响应单元

响应单元：对分析结果作出反应

事件数据库：存放各种中间和最终数据

##### **入侵检测技术**

**误用检测技术**\
基于模式匹配原理。收集非正常操作的行为特征，建立相关的特征库。

特点：误报低、漏报高，无法检测新的入侵

**异常检测技术**\
基于统计分析原理。首先总结正常操作应该具有的特征（用户轮廓），试图用定量的方式加以描述，当用户活动与正常行为有重大偏离时即被认为是入侵。

### DHCP服务及其配置

DHCP(Dynamic Host Configuration Protocol，动态主机配置协议）

在没有DHCP中继的情况下，一个DHCP客户端（即请求IP地址的设备）只能向同一子网内的DHCP服务器发送请求。如果客户端和服务器不在同一个广播域（即它们之间的网络被路由器分隔开），那么客户端的DHCP广播请求就无法到达服务器。

DHCP中继代理（通常称为DHCP Relay Agent或DHCP Helper）的作用是接收来自客户端的DHCP广播消息，并将这些消息转发到其他子网中的DHCP服务器。它充当了一个中介或中继的角色

### 内网环境

工作组:无法控制工作组的朋友（计算机教室的各个主机）

用来管理服务器及客户端计算机账户，包含组策略（统一所有电脑同一背景之类）等等。

**安全域的划分** 1.内网（安全级别最高）：分为核心区（存储企业最重要的数据，只有很少的主机能够访问）和办公区（员工日常工作区，一般能够访问DMZ，部分主机可以访问核心区）

2.DMZ（Demilitarized Zone，边界网络，隔离区，安全级别中等）：作为内网中安全系统和非安全系统之间的缓冲区，用于对外提供服务，一般可以放置一些必须公开的服务器设施

### 网络适配器

通常称为网卡，是计算机硬件或软件的一部分，用于计算机之间通过有线或无线网络进行**通信**。以下是一些常见的网络适配器类型及其作用：

1.  **以太网卡**：

    *   作用：提供有线网络连接，使用以太网标准传输数据。

    *   常见接口：RJ-45端口，使用双绞线电缆连接。

2.  **无线网卡**：

    *   作用：提供无线网络连接，使用Wi-Fi标准连接到无线网络。

    *   常见标准：802.11 a/b/g/n/ac/ax等。

3.  **蓝牙适配器**：

    *   作用：通过蓝牙技术提供短距离无线连接。

    *   用途：连接蓝牙耳机、键盘、鼠标等外围设备。

4.  **光纤网卡**：

    *   作用：通过光纤线缆提供高速网络连接。

    *   用途：数据中心、高速互联网连接等。

5.  **虚拟网络适配器**：

    *   作用：在虚拟化环境中，由虚拟化软件创建，用于虚拟机的网络连接。

    *   用途：宿主机与虚拟机之间的网络通信，或虚拟机与外部网络之间的通信。

6.  **USB网络适配器**：

    *   作用：提供便携式网络连接，通过USB接口连接到计算机。

    *   用途：便携式以太网或无线网络连接。

7.  **串行和并行端口**：

    *   作用：虽然不是现代网络通信的主流方式，但串行和并行端口曾被用于某些特定的网络连接。

8.  **卫星网络适配器**：

    *   作用：通过卫星连接提供远程或移动网络连接。

    *   用途：移动车辆、船舶、飞机或偏远地区。

9.  **移动网络（蜂窝）适配器**：

    *   作用：提供移动网络连接，如4G/5G。

    *   用途：移动设备或便携式热点。

10. **VPN适配器**：

    *   作用：在软件层面，提供虚拟私人网络连接，通过加密技术连接到远程网络。

    *   用途：远程访问公司或组织的内部网络。

每种网络适配器都有其特定的应用场景和优势。例如，以太网卡因其稳定性和速度而被广泛用于局域网（LAN）环境，而无线网卡则提供了移动性和灵活性，允许用户在没有物理线缆的情况下连接网络。**虚拟网络适配器在虚拟化技术中非常重要，它们允许虚拟机像真实机器一样参与网络通信。**

**虚拟网卡**

1.  **VMnet1 (Host-Only Networking)**：

    *   在仅主机模式下，`vmnet1` 适配器被用来创建一个虚拟网络，它只包括宿主机和连接到该网络的所有虚拟机。

    *   虚拟机通过这个适配器可以与宿主机通信，但不能直接访问外部网络。

    *   这种模式适用于需要宿主机访问虚拟机，但不需要或不希望虚拟机访问外部网络的情况。

2.  **VMnet8 (NAT Networking)**：

    *   `vmnet8` 适配器通常用于**NAT模式**，允许虚拟机通过宿主机的IP地址访问外部网络。

    *   在这种模式下，虚拟机共享宿主机的公网IP地址，并通过NAT引擎进行网络地址转换。

    *   虚拟机可以访问互联网，但外部网络不能直接访问虚拟机。

网桥（Bridge）是一种网络设备，用于连接两个或多个网络段，并对数据包进行转发，有物理上的也有虚拟的。

**桥接模式**可以看作是在虚拟化环境中实现网桥功能的一种方式。通过桥接模式，虚拟机能够直接接入物理网络，就像它们是通过物理网桥连接的一样。

在虚拟化软件中，如VMware或VirtualBox，桥接模式通常通过虚拟网络适配器来实现，这些适配器充当虚拟网桥，将虚拟机的网络流量桥接到宿主机的物理网络接口上。

### 遇到下载的三种方式

正常：直链：一个人分享资源，上传到网盘服务器给大家下载，但是服务器的带宽是有限的，只有一条链路的话会很拥塞，下载速度很慢。

##### **bittorrent协议的下载**

&#x20;基于p2p（点对点网络）的方式，分享者制作分享文件，然后多人下载的时候**一人下载一个部分**最后各自分享给对方合起来(也就是使用每个主机的资源下载各个部分)，不用经过之前的网盘服务器.

&#x20;需要经过tracker服务器(找到具体是哪些人正在下载这个东西)。

**过程：**

用户需要首先下载一个种子文件（**.torrent** 文件），该文件包含了Tracker服务器的信息和文件的哈希值。用户通过种子文件连接到Tracker服务器，服务器再提供其他下载者的IP地址，从而实现文件块的交换

**缺点**：容易遇到死种，下载不了

**经典工具**：bitcomet

但是由于tracker容易被禁(版权)，所以后面的版本让每个人都变成了tracker服务器，原理是每个人在下载资源的同时也要上传分享自己的资源。表现就是**磁力链接**，起到了去中心化的效果(每个人都是tracker服务器)

在数据传输的过程中需要考虑三个特性。 第一个机密性，不能让别人知道我的内容是什么。 第二个是完整性，不能保证数据被增加或者篡改。 第三个是端点鉴别，通信双方要确认另一方身份 ​于是出现了PKI（public key infraction）公钥基础设施用来保证信息三个特性 ​内容： ​1.通信双方使用同一个密钥进行加解密，相当于把你的信息放到一个箱子里，有钥匙的人才能打开。但是这个要双方“线下”或者及其安全的地方才能打开。使用同一个钥匙，所以这种方式叫做对称加密技术，解决了机密性。 ​2.通信双方使用一个特殊的🔒 ，里面有两个孔，一个插的是公钥，一个是私钥，两把都是钥匙，其中一个锁了🔒 只有另一个才能打开。 由于公钥是公共的，全网都有，在发送的过程中，发送方可以先用一种加密方式（密钥）把信息变成密文，然后拿公钥把这个密钥再锁起来变成密钥信封，再把密文和信封发给接收方，接受方拿私钥解密信封得到密钥，再拿密钥解密密文，结果也就出来了（简单来说就是加解密两次），这就是非对称加密技术（不是拿同一个🔑 ）但是这里有个弊端就是，接收方的信息可以被别人截取下来随便篡改，所以不好。于是延伸出了下面的技术。 ​3.使用发送方的私钥加密，然后公钥解密。发送方先用自己的私钥加密信息，接收方拿公钥解密，这个时候由于私钥是保密的，别人篡改不了消息，就保证了数据的完整性，并且由于私钥只有接收方有，相当于一种“签名”，接收方也就能根据得到的信息辨别出是不是黑客了 ​4.但是这依旧有危险，中间人可以截取你发出来的私钥解密以后再自己利用信息自己搞一串公钥私钥和接收方进行对话。所以就有了CA可靠机构，它会给你的公钥或者私钥（基于你的非对称使用方式）一个数字证书，这个就是接受方用来验证你的身份的，这也是数字签名技术，到这里基本上三个特性就解决了。

### 渲染

渲染HTML文件是指将HTML代码转换成用户在浏览器中可以看到的网页的过程。这个过程涉及到以下几个步骤：

1.  **解析HTML**：浏览器首先读取HTML文件，解析其中的标签和属性，构建一个DOM（文档对象模型）树。

2.  **解析CSS**：浏览器接着解析与HTML文件关联的CSS样式表，确定页面元素的样式和布局。

3.  **构建渲染树**：将DOM树和CSS样式结合起来，创建一个渲染树，这个树包含了所有可见的元素和它们的样式信息。

4.  **布局**：浏览器根据渲染树计算每个元素在页面上的位置和大小。

5.  **绘制**：最后，浏览器将渲染树中的元素绘制到屏幕上，形成用户可以看到的网页。

这个过程通常是由浏览器的渲染引擎自动完成的。常见的浏览器渲染引擎包括Chrome的Blink、Firefox的Gecko、Safari的**WebKit**等。

jinja2等等模板一般就使用在渲染的时候

### **云服务**

云计算诞生之前，大部分计算资源是处于“裸金属”状态的物理机，运维人员选择对应规格的硬件，建设机房的 IDC 网络，完成服务的提供，投入硬件基础建设和维护的成本很高。云服务出现之后，使用者可以直接购买云主机，基础设施由供应商管理，这种方式也被称作 IaaS (Infrastructure-as-a-Service) 。

随着架构的继续发展，应用的运行更加细粒度，部署环境容器化，各个功能拆成微服务或是Serverless的架构。

### **Serverless**

Serverless 架构由两部分组成，即 Faas (Function-as-a-Service) 和 BaaS (Backend-as-a-Service) 。

FaaS是运行平台，用户上传需要执行的逻辑函数如一些定时任务、数据处理任务等到云函数平台，配置执行条件触发器、路由等等，就可以通过云平台完成函数的执行。

BaaS包含了后端服务组件，它基于 API 完成第三方服务，主要是数据库、对象存储、消息队列、日志服务等等。

### **微服务**

微服务起源于2005年Peter Rodgers博士在云端运算博览会提出的微Web服务 (Micro-Web-Service)，根本思想类似于Unix的管道设计理念。2014年，由Martin Fowler与 James Lewis共同提出了微服务的概念，定义了微服务架构风格是一种通过一套小型服务来开发单个应用的方法，每个服务运行在自己的进程中，并通过轻量级的机制进行通讯 (HTTP API) 。

微服务是一种应用于组件设计和部署架构的软件架构风格。它利用模块化的方式组合出复杂的大型应用程序：

*   各个服务功能内聚，实现与接口分离。

*   各个服务高度自治、相互解耦，可以独立进行部署、版本控制和容量伸缩。

*   各个服务之间通过 API 的方式进行通信。

*   各个服务拥有独立的状态，并且只能通过服务本身来对其进行访问。

随着微服务技术的不断发展，这种思想也被应用到了前端。2018年，第一个微前端工具single-spa出现在github。而后出现了基于single-spa的框架qiankun。

### svchost.exe

svchost.exe 是从动态链接库 (DLL) 中运行的服务的通用主机进程名称。

### 服务和进程的关系

**进程**是操作系统中执行程序的**实例**，它是一个动态的实体，具有独立的内存空间，它可以**创建线程**，线程是进程中的一个执行单元，是CPU调度和分派的基本单位，进程可以是用户交互的，也可以是后台运行的，**服务**是一种**特殊**类型的系统**进程**，它们通常在系统启动时自动运行，或者由系统管理员手动启动。服务通常不提供用户界面，它们在**后台运行**，**为系统或应用程序提供支持**。服务可以是**系统**服务，也可以是**第三方**服务，它们通常用于执行系统级任务

1.  `$wsdl`（必需）：

    *   这是一个混合类型的参数，它可以是一个包含 WSDL 文件 URL 的字符串，或者是一个包含 WSDL 文档内容的字符串。

    *   WSDL（Web Services Description Language）是一个 XML 文件，它描述了 SOAP 服务的公共接口，包括服务中可用的方法、参数和返回值的详细描述。

    *   如果设置为 `null` 或 `false`，则表示客户端将不使用 WSDL 文件，而是使用非 WSDL 模式。

2.  `$options`（可选）：

    *   这是一个关联数组，用于设置 `SoapClient` 对象的各种选项。这些选项可以用来自定义客户端的行为，例如设置 SOAP 头部、代理服务器信息、认证信息等。

    *   常见的选项包括：

        *   `"location"`：SOAP 服务的 URL，即使在使用 WSDL 时，也可以通过这个选项来覆盖 WSDL 文件中指定的 URL。

        *   `"uri"`：服务的目标命名空间，通常在 WSDL 文件中指定。

        *   `"style"`：可以是 `SOAP_RPC` 或 `SOAP_DOCUMENT`，指定 SOAP 的风格。

        *   `"use"`：可以是 `SOAP_ENCODED` 或 `SOAP_LITERAL`，指定 SOAP 的使用类型。

        *   `"soap_version"`：可以是 `SOAP_1_1` 或 `SOAP_1_2`，指定 SOAP 的版本。

        *   `"trace"`：布尔值，如果设置为 `true`，则会显示 SOAP 请求和响应的调试信息。

        *   `"exceptions"`：布尔值，如果设置为 `true`，则客户端会在遇到 SOAP 错误时抛出异常。

        *   `"cache_wsdl"`：指定 WSDL 文件的缓存机制，可以是 `WSDL_CACHE_NONE`、`WSDL_CACHE_DISK` 或 `WSDL_CACHE_MEMORY`。
